# 徽章改 emoji 圆章：弃用真实边界轮廓

日期：2026-07-09。背景：州视图徽章原按公园真实边界形状绘制（ADR-0001 布局 + park-bounds.js 122KB 静态边界数据 + SVG clip 刮层）。用户反馈轮廓不好看，要求直接显示 emoji；并要求每园 emoji 准确、63 座不重复。

## 决策

1. **徽章 = emoji 圆章**：沿用原 medallion 兜底样式（圆盘 + emoji + 虚线环 + 圆形刮层），从"兜底"升为唯一形态；尺寸随州内布局密度缩放（44–96px）。刮开机制（Scratchable、长按就绪、刮满判定）不变，只是 clip 从边界形变回圆形。
2. **63 园 emoji 全集唯一**：审计出 9 组重复（24 座），重排 16 座。选取原则：取该园最具辨识度的单个意象（🐐 冰川山羊、🦬 黄石野牛、👑 国王峡谷、⛏️ 兰格尔铜矿、🌌 大盆地暗夜、♨️→🫧 拉森地热泥泉等），保持单 emoji（组合在 40px 下太挤）。放弃自绘 63 个 SVG 图标：维护成本高，emoji 辨识度已够。
3. **park-bounds.js 停止加载**（parks.html 去掉 script、SW 预缓存剔除，省 122KB 首载）；**文件保留在仓库**——含优胜美地边界数据（永不删约束），且未来想恢复轮廓玩法可直接接回。
4. 全美视图公园标记用同一套 emoji，自动受益于唯一化。

## Consequences

- 首载体积 -122KB；徽章渲染不再依赖 d3 投影计算，代码 -~60 行（fcShape/makeFigure/fetchBoundary 删除）。
- PARKS 数据 em 字段现在承担"唯一标识视觉"职责：新增/修改公园时必须保持全集不重复。
- 新 emoji（🫎 驼鹿、🫧 泡泡、🦬 野牛）需要较新系统字体（≈2021–2023 Unicode），旧设备可能显示豆腐块——目标用户设备（iOS 17+）无碍。
