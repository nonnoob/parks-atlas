# 图鉴云存档：加密整包进 GitHub 仓库，Cloudflare Worker 代写

日期：2026-07-09。背景：打卡记录只存 localStorage，换设备即空图鉴；口令只派生签名身份，救不回记录。用户要求：打卡后数据远程持久化，且与口令唯一绑定（任何设备输口令即恢复）；分享维持只读视图。约束：GitHub Pages 静态站无后端（ADR-0005），页面里不能放 GitHub token（公开泄露即被 GitHub 自动吊销，且等于任何人可写仓库）。

## 决策

1. **存档进 nonnoob/trip 仓库 `data/atlas/<查找键>.json`**。查找键 = PBKDF2(口令, 固定盐 `np-cloud-v1`)，任何设备同口令算出同一键——这就是"口令唯一绑定"的实现；也意味着**口令即图鉴**：新设备输入口令，云端有档即恢复，无档即新建。
2. **整包加密再上传**：身份 + 全部打卡记录 JSON 用口令派生的 AES-GCM 加密（与锁私钥同一把钥匙、不同 IV），公开仓库里看不到去过哪些公园。外层只暴露公钥、编号、盐、密文、以及**对密文的 ECDSA 签名**。
3. **写入经 Cloudflare Worker 代理**（tools/cloud-worker.js）：GitHub fine-grained PAT（仅 trip 仓库 Contents 读写）只存 Worker Secret，不落任何设备。Worker 验签后才代写：首写登记公钥（TOFU），此后必须同一公钥签名才能覆盖——没有口令的人既解不开也盖不掉别人的存档。公钥不匹配返回 409。
4. **同步策略求简**：打卡/取消/新建后 1.2 秒防抖推送；解锁时拉云档对账——云端多的记录并进本地（逐条验签），本地多的回传。失败标 np_dirty，下次动作重试。不做实时/冲突 UI。
5. **分享不变**：仍是 #share= 只读快照链接（ADR 前的既有机制），不带口令、不指向云档。
6. 放弃的路：PAT 存设备 localStorage（朋友拿口令打卡无法持久化）；token 混淆后内嵌（GitHub secret scanning 直接吊销）；第三方存储服务（隐私与依赖）。

## Consequences

- 新增站外基建：Cloudflare Worker（免费档）+ 用户自管 PAT，凭据全程用户自己填。Worker 失联时 App 照常可用，只是不同步。
- 每次推送 = trip 仓库一次 commit，会触发 Pages 重建；打卡频率极低（63 座封顶）+ 防抖，不构成压力。
- bundle 格式（v/id/fp/pub/salt/iv/ct/sig）与查找键派生参数**不可变**：已上传的存档依赖它。
- 口令撞车（两人恰好同口令）：后者恢复时解不开密文会被拒；后者新建身份推送会收 409。概率忽略；提示换口令即可。
- app-parks.js 的 CLOUD 常量为空即整套静默关闭，回滚 = 置空重发。
